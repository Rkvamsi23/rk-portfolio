Cleaning 2M Seller Records

Optiwise Python SQL

Problem

Optiwise collects large JSON dumps from marketplaces like Amazon and Walmart. To do any serious analytics on seller performance we needed a clean, relational view instead of nested, inconsistent JSON.

My role

  • Wrote Python scripts to parse and validate seller JSON for thousands of records at a time
  • Designed and refined SQL tables for sellers, products, brands and categories
  • Migrated more than two million seller records into the new schema
  • Implemented checks to catch missing or inconsistent keys before load

Approach

  • Identified stable fields across different JSON versions, such as seller id, marketplace, rating, review count and category paths
  • Flattened nested structures into separate tables with foreign keys
  • Used incremental loading so that new crawls could append without breaking existing data

Impact

  • Reduced query time for common analytics questions by moving from raw JSON scans to indexed SQL tables
  • Enabled dashboards that show seller distribution, rating patterns and share of catalogue at scale
  • Gave the team a foundation to build more advanced models for seller quality and opportunity

Skills and tools

  • Python, Pandas
  • SQL and relational schema design
  • Data validation and migration for large datasets

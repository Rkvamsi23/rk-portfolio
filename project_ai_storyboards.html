Teaching Algorithms To Write Ads

LLMs A/B testing marketing startup

Goal

Help brands generate short form video scripts at scale while understanding what kinds of emotional narratives the system encourages when everything is optimised for performance.

System overview

  • Input product information and brand voice guidelines
  • Use a large language model to generate hooks, scripts and simple scene descriptions for Reels or Shorts
  • Push multiple variants into campaigns
  • Collect metrics such as view through rate and click through rate
  • Feed the results back into prompt templates so that later scripts are tuned by what worked before

My role

  • Worked with the product team to define what information each script needs
  • Designed tables for storing generated scripts, variants and campaign metrics
  • Built Python utilities to clean and join performance data with script metadata
  • Analysed which hooks and framings correlated with better metrics and shared patterns with writers and engineers

Tools and methods

  • Python and Pandas for data cleaning and analysis
  • SQL for querying campaign data
  • A/B testing to compare different hook strategies

What we saw

  • Hooks built around urgency, discounts or strong claims generally outperformed slower, more subtle stories
  • Scripts that left room for ambiguity or build up underperformed, at least on the short metrics we tracked
  • Over time this pushed the system toward a narrow emotional range: high energy, fast moving, low ambiguity

Why it matters

The work made me see these tools not just as creative assistants but as infrastructures that quietly shape which emotional experiences are cheap and scalable. This observation now sits at the centre of my research questions about humour and algorithmic curation.
